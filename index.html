Nexe

Create a single executable out of your node.js apps

https://github.com/nexe/nexe

  • Mention pkg as an alternative
  • Nexe embeds native modules, pkg does not.
  • Nexe supports input via stdin as well as command line parameters.

package.json 


						// ...
						"scripts": {
							// ...
							"build": "npx nexe --build index.js -o bin/elise",
							// ...
						}
					

						$ npm run build

Nexe Benefits

  • Single-file deployment
  • Includes embedding native modules (*.node files)
  • Bundles known working version of the Node runtime with your application
  • Runtime can be upgraded anytime during a normal deployment

Cache Configuration 


						cache:
						  untracked: true
						  key: "$CI_COMMIT_REF_NAME"
						  paths:
						    - node_modules/
  • Artifacts are a better alternative as caches are limited per runner.
  • Mention GitLab CI variables - $CI_COMMIT_REF_NAME refers to the current commit hash.
  • Caches are removed after every job and copied into the workspace at the beginning of the next job.
